# Как сообщить о баге

neurogate сейчас в раннем доступе — баги ожидаемы. Чтобы воспроизвести и починить, нужны минимальные исходные данные. Без них мы не можем отделить баг шлюза от проблемы провайдера / соединения / клиента.

**Куда писать:** GitHub Issue в этой репе. Шаблон формы — `.github/ISSUE_TEMPLATE/bug_report.md` подгрузится автоматически при «New issue».

## Чек-лист (минимум)

- [ ] **Что произошло** — одна-две фразы. Что должно было быть, что получилось вместо.
- [ ] **Текст ошибки целиком** — копи-паст, не пересказ. Если приложение режет текст — скрин консоли / network tab из DevTools.
- [ ] **Когда произошло** — точное время с часовым поясом (например, `2026-05-09 14:32 UTC+3`). Нужно чтобы найти запись в логах.
- [ ] **Что делал до ошибки** — последовательность действий. Например: «открыл Cursor → задал вопрос про код → пошёл стрим, на 3-й секунде оборвался». **Воспроизводится повторно или один раз?**
- [ ] **Какой клиент** и его версия — Cursor / Claude Code / Continue / Cline / Aider / curl / собственный скрипт / браузер; версия (например, `Cursor 1.4.2`).
- [ ] **Какая модель / chain** была запрошена — поле `model:` в настройках клиента. Например: `auto`, `chat`, `deepseek:reasoner`.
- [ ] **Версия neurogate** — `git rev-parse --short HEAD` или дата клонирования.

## Желательно (если можешь)

- **Request ID** — `x-request-id` в headers ответа. По нему найдём конкретный запрос в логах за миллисекунды.
- **Скрин Network tab** или вывод `curl -v`.
- **Размер промпта** — короткий вопрос или длинный документ. Длинный context часто выявляет баги, которых нет на коротком.
- **Stream или non-stream**.
- **Тип соединения и оператор/ISP** — особенно если ты в РФ или живёшь за DPI/блокировками. Ниже про это подробнее.

## Регионы и провайдеры (подсказка)

neurogate за собой держит 15+ upstream-провайдеров (Groq, Cerebras, SambaNova, Gemini, OpenRouter, DeepSeek, …). Часть из них **блокируется или режется на уровне отдельных ISP** — эта блокировка не от шлюза, а от твоего провайдера. Признаки: `connection reset` / `timeout` / `SSL handshake fail` на конкретный домен.

Некоторые наблюдения с прошлых отчётов (РФ):

- МТС иногда режет HuggingFace и часть Cloudflare-эндпоинтов.
- Мегафон блокирует часть Google IP-блоков → Gemini нестабильна.
- Ростелеком блокирует часть международного трафика по DPI.
- Мобильный интернет временами имеет странный SNI-фильтр.

Без информации про оператора мы можем час крутить логи в поисках бага в коде, которого нет.
